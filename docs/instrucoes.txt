Documento de Instruções: Agentic Autonomous Organization (AAO)
============================================================

Este documento serve como guia de base tanto para DESENVOLVEDORES HUMANOS quanto para AGENTES DE INTELIGÊNCIA ARTIFICIAL que interagirem com este repositório.

1. OBJETIVO DO REPOSITÓRIO
Este repositório front-end é a face pública do ecossistema Agentic Autonomous Organization (AAO). Ele é responsável por explicar e expor as métricas e capacidades dos nossos agentes autônomos. 
O projeto foca em delegar fluxos de trabalho (Code Review, Automação de PRs, Dependências) para modelos de IA como GPT, Claude e Gemini.

2. INSTRUÇÕES PARA AGENTES DE IA (LLMs, Copilots, etc)
- A paleta de cores deve ser estritamente mantida: Minimalista Dark Mode (Preto puro absoluto) com acentos vítreos. NUNCA mude para temas claros.
- Estilização: Utilizar **Tailwind CSS** para classes utilitárias, complementado com CSS puro (`index.css` e `Navbar.css`) apenas para efeitos avançados de Liquid Glass e animações.
- Animações: NUNCA utilizar bibliotecas pesadas como framer-motion ou react-spring. O projeto preza pela performance 100/100 no PageSpeed usando exclusivamente transições CSS Nativas ultra-otimizadas.
- O Layout segue um modelo de "Apresentação em Slides Horizontais", gerenciado via React state puro (`App.tsx`) ativado por eventos nativos de `wheel` do mouse.
- Sempre incremente a versão usando a nomenclatura "vX.Y.Z" (Semantic Versioning) no instrucoes.txt, README e nas mensagens de commit.
- OBRIGATÓRIO: Leia e siga o arquivo `docs/AI_WORKFLOW.txt` antes de qualquer alteração. Ele define o protocolo completo de versionamento e regras arquiteturais.

3. INSTRUÇÕES PARA DESENVOLVEDORES HUMANOS
- Ambiente: Vite + React + TypeScript + Tailwind CSS.
- Execução local: `npm install` seguido de `npm run dev`.
- Arquitetura de Performance: Toda renderização pesada (Aurora e DotField) está protegida por `React.memo` para evitar recálculos durante a navegação. O bundle principal foi otimizado para possuir < 65KB.
- As configurações de CI/CD para o GitHub Pages estão no `.github/workflows/deploy.yml`. O roteamento (base url) no `vite.config.ts` está configurado para o root `/`.
