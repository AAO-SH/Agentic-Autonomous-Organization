Instructions Document: Agentic Autonomous Organization (AAO)
============================================================

This document serves as a base guide for both HUMAN DEVELOPERS and ARTIFICIAL INTELLIGENCE AGENTS interacting with this repository.

1. REPOSITORY OBJECTIVE
This front-end repository is the public face of the Agentic Autonomous Organization (AAO) ecosystem. It is responsible for explaining and exposing the metrics and capabilities of our autonomous agents. 
The project focuses on delegating workflows (Code Review, PR Automation, Dependencies) to AI models like GPT, Claude, and Gemini.

2. INSTRUCTIONS FOR AI AGENTS (LLMs, Copilots, etc)
- The color palette must be strictly maintained: Minimalist Dark Mode (Absolute pure black) with vitreous accents. NEVER change to light themes.
- Styling: Use **Tailwind CSS** for utility classes, complemented with pure CSS (`index.css` and `Navbar.css`) only for advanced Liquid Glass effects and animations.
- Animations: NEVER use heavy libraries like framer-motion or react-spring. The project values 100/100 performance on PageSpeed using exclusively ultra-optimized Native CSS transitions.
- The Layout follows a "Horizontal Slides Presentation" model, managed via pure React state (`App.tsx`) activated by native mouse `wheel` events.
- Always increment the version using the "vX.Y.Z" (Semantic Versioning) nomenclature in instrucoes.txt, README, and commit messages.
- MANDATORY: Read and follow the `docs/AI_WORKFLOW.txt` file before any changes. It defines the complete versioning protocol and architectural rules.

3. INSTRUCTIONS FOR HUMAN DEVELOPERS
- Environment: Vite + React + TypeScript + Tailwind CSS.
- Local execution: `npm install` followed by `npm run dev`.
- Performance Architecture: All heavy rendering (Aurora and DotField) is protected by `React.memo` to prevent recalculations during navigation. The main bundle was optimized to be < 65KB.
- CI/CD settings for GitHub Pages are in `.github/workflows/deploy.yml`. Routing (base url) in `vite.config.ts` is configured for the root `/`.
